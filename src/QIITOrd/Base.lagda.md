# Ordinals as a quotient inductive-inductive type

This module is the foundation of the library. It defines the type `Ord` of
ordinals вҖ” Brouwer trees вҖ” as a **quotient inductive-inductive type (QIIT)**,
*mutually* with its order `_вүӨ_`, following Kraus, Nordvall Forsberg and Xu,
[*Type-Theoretic Approaches to Ordinals*](https://arxiv.org/abs/2208.03844). It
also derives the two eliminators (`elimProp`, `вүӨ-elimProp`) that every later
module is built from, and the basic order lemmas.

```agda
{-# OPTIONS --cubical --safe #-}

module QIITOrd.Base where

open import Cubical.Foundations.Prelude
open import Cubical.Foundations.Function
open import Cubical.Foundations.HLevels
open import Cubical.Foundations.Path
open import Cubical.Data.Nat as в„• using (в„•)
open import Cubical.Data.Sigma
open import Cubical.Relation.Nullary using (В¬_)
open import Cubical.Relation.Binary.Base using (Rel; module BinaryRelation)
open BinaryRelation using (isRefl'; isTrans')
```

We reuse the cubical library's binary-relation vocabulary: `Rel A B в„“` is the
type of `в„“`-valued relations, and `BinaryRelation`'s `isRefl'` / `isTrans'` are
the (implicit-argument) reflexivity and transitivity predicates. We only ever
need homogeneous relations `Rel A A в„“`.

## Sequences and bisimulation

A limit ordinal is built from a sequence `в„• вҶ’ A`. Two such sequences should give
the *same* limit exactly when they are mutually cofinal вҖ” they **bisimulate**.
`f вүј g` says every `f n` is dominated by some `g m` (the existence is a
propositional truncation `вҲғ`), and `f вүҲ g` is bisimilarity: `f вүј g` and `g вүј f`.
This is stated generally for any relation `_вүӨ_`, so the same definitions serve
both the order on ordinals and, later, the order on order-codes.

```agda
module Bisimulation {в„“ в„“'} {A : Type в„“} (_вүӨ_ : Rel A A в„“') where
  private Seq = в„• вҶ’ A

  _вүјвҹЁ_вҹ©_ : Seq вҶ’ в„• вҶ’ Seq вҶ’ Type _
  f вүјвҹЁ n вҹ© g = вҲғ[ m вҲҲ в„• ] f n вүӨ g m

  _вүј_ : Rel Seq Seq _
  f вүј g = вҲҖ n вҶ’ f вүјвҹЁ n вҹ© g

  _вүҲ_ : Rel Seq Seq _
  f вүҲ g = f вүј g Г— g вүј f
```

A sequence is **monotone** when `suc (f n) вүӨ f (suc n)`, i.e. it strictly
increases. A `MonoSeq` bundles a sequence with that proof, and `f [ n ]` projects
out its `n`-th term. Limits will be taken only of monotone sequences.

```agda
module MonotonicSequence {в„“ в„“'} {A : Type в„“} (suc : A вҶ’ A) (_вүӨ_ : Rel A A в„“') where
  private Seq = в„• вҶ’ A

  isMonoSeq : Seq вҶ’ Type _
  isMonoSeq f = вҲҖ n вҶ’ suc (f n) вүӨ f (в„•.suc n)

  MonoSeq : Type _
  MonoSeq = ОЈ[ f вҲҲ Seq ] isMonoSeq f

  infix 30 _[_]
  _[_] : MonoSeq вҶ’ в„• вҶ’ A
  f [ n ] = fst f n
```

## The QIIT definition

`Ord` and `_вүӨ_` are declared *mutually*: `_вүӨ_` already appears in the type of
`lim` (through `MonoSeq`), and `Ord` appears in the type of `_вүӨ_`. We first
forward-declare both, plus a private copy `pre-suc` of the successor (needed to
instantiate `MonotonicSequence` before `suc` itself is in scope), then open the
two helper modules at `_вүӨ_` so that `MonoSeq`, `_вүј_`, `вүҲ`, `_[_]` are available.

```agda
data Ord : Type
data _вүӨ_ : Rel Ord Ord в„“-zero
private pre-suc : Ord вҶ’ Ord

open Bisimulation _вүӨ_ public
open MonotonicSequence pre-suc _вүӨ_ public

variable
  Оұ ОІ Оі : Ord
  f g h : MonoSeq
  n : в„•
```

`Ord` has the three ordinal formers вҖ” `zero`, `suc`, and `lim` of a monotone
sequence вҖ” and two *higher* constructors that make it a QIIT: `lвүЎl` quotients
bisimilar limits to be equal (the path constructor), and `isSetOrd` truncates
`Ord` to a set, so all of its path structure is determined by these.

```agda
data Ord where
  zero     : Ord
  suc      : Ord вҶ’ Ord
  lim      : MonoSeq вҶ’ Ord
  lвүЎl      : fst f вүҲ fst g вҶ’ lim f вүЎ lim g
  isSetOrd : isSet Ord
```

The order `_вүӨ_` is generated by: `zero` is least (`zвүӨ`); `suc` is monotone
(`sвүӨs`); `lim f` is an *upper bound* of its sequence and the *least* one (`вүӨl`
and `lвүӨ`); together with transitivity and the proof that `_вүӨ_` is propositional.
The last two are constructors because `_вүӨ_` is defined simultaneously with `Ord`,
so we cannot derive them after the fact.

```agda
data _вүӨ_ where
  zвүӨ      : zero вүӨ Оұ
  sвүӨs     : Оұ вүӨ ОІ вҶ’ suc Оұ вүӨ suc ОІ
  вүӨl      : Оұ вүӨ f [ n ] вҶ’ Оұ вүӨ lim f
  lвүӨ      : (вҲҖ n вҶ’ f [ n ] вүӨ Оұ) вҶ’ lim f вүӨ Оұ
  вүӨ-trans : isTrans' _вүӨ_
  isPropвүӨ : isProp (Оұ вүӨ ОІ)

pre-suc = suc
```

This completes the QIIT.

## Transfinite induction into propositions

The work-horse recursor `elimProp` is structural recursion on `Ord` into a
**proposition-valued** family `P`. Because each `P Оұ` is a proposition, the two
higher constructors are discharged automatically: the `lвүЎl` case by
`isPropвҶ’PathP` and the `isSetOrd` case by `isPropвҶ’SquareP`. The user only
supplies the `zero`/`suc`/`lim` methods вҖ” and in the `lim` method the inductive
hypothesis arrives **already unpacked** as `вҲҖ n вҶ’ P (f [ n ])`. This unpacking is
what later lets transitivity of the order-code be defined structurally.

```agda
elimProp : вҲҖ {в„“} {P : Ord вҶ’ Type в„“} вҶ’
  (вҲҖ Оұ вҶ’ isProp (P Оұ)) вҶ’
  P zero вҶ’
  (вҲҖ {Оұ} вҶ’ P Оұ вҶ’ P (suc Оұ)) вҶ’
  (вҲҖ {f} вҶ’ (вҲҖ n вҶ’ P (f [ n ])) вҶ’ P (lim f)) вҶ’
  (Оұ : Ord) вҶ’ P Оұ
elimProp Pprop Pz Ps Pl zero    = Pz
elimProp Pprop Pz Ps Pl (suc Оұ) = Ps (elimProp Pprop Pz Ps Pl Оұ)
elimProp Pprop Pz Ps Pl (lim f) = Pl О» n вҶ’ elimProp Pprop Pz Ps Pl (fst f n)
elimProp Pprop Pz Ps Pl (lвүЎl {f} {g} fвүҲg i) = isPropвҶ’PathP
  (О» j вҶ’ Pprop (lвүЎl fвүҲg j))
  (Pl О» n вҶ’ elimProp Pprop Pz Ps Pl (fst f n))
  (Pl О» n вҶ’ elimProp Pprop Pz Ps Pl (fst g n)) i
elimProp Pprop Pz Ps Pl (isSetOrd _ _ p q i j) = isPropвҶ’SquareP
  (О» i j вҶ’ Pprop $ isSetOrd _ _ _ _ i j) refl refl
  (О» i вҶ’ elimProp Pprop Pz Ps Pl (p i))
  (О» i вҶ’ elimProp Pprop Pz Ps Pl (q i)) i j
```

Dually, `вүӨ-elimProp` is structural recursion on a **derivation** of `Оұ вүӨ ОІ` into
a proposition-valued family. Its `lвүӨ` method is the subtle one: it supplies, in
addition to the induction hypotheses on the proofs `f [ n ] вүӨ Оұ`, the sequence's
*monotonicity* hypotheses `вҲҖ n вҶ’ P (snd f n)`. That extra hypothesis is exactly
what `encodeвүӨ` needs later, and is why a plain "induction on the derivation" is
not enough.

```agda
вүӨ-elimProp : вҲҖ {в„“} {P : вҲҖ {Оұ ОІ} вҶ’ Оұ вүӨ ОІ вҶ’ Type в„“} вҶ’
  (вҲҖ {Оұ ОІ} (ОұвүӨОІ : Оұ вүӨ ОІ) вҶ’ isProp (P ОұвүӨОІ)) вҶ’
  (вҲҖ {Оұ} вҶ’ P (zвүӨ {Оұ})) вҶ’
  (вҲҖ {Оұ ОІ} (ОұвүӨОІ : Оұ вүӨ ОІ) вҶ’ P ОұвүӨОІ вҶ’ P (sвүӨs ОұвүӨОІ)) вҶ’
  (вҲҖ {Оұ f n} (ОұвүӨf : Оұ вүӨ f [ n ]) вҶ’ P ОұвүӨf вҶ’ P (вүӨl {f = f} ОұвүӨf)) вҶ’
  (вҲҖ {f Оұ} (fвүӨОұ : вҲҖ n вҶ’ f [ n ] вүӨ Оұ) вҶ’ (вҲҖ n вҶ’ P (fвүӨОұ n)) вҶ’ (вҲҖ n вҶ’ P (snd f n)) вҶ’ P (lвүӨ {f} fвүӨОұ)) вҶ’
  (вҲҖ {Оұ ОІ Оі} {ОұвүӨОІ : Оұ вүӨ ОІ} {ОІвүӨОі : ОІ вүӨ Оі} вҶ’ P ОұвүӨОІ вҶ’ P ОІвүӨОі вҶ’ P (вүӨ-trans ОұвүӨОІ ОІвүӨОі)) вҶ’
  (ОұвүӨОІ : Оұ вүӨ ОІ) вҶ’ P ОұвүӨОІ
вүӨ-elimProp Pprop Pz Ps Pu Pl Pt zвүӨ          = Pz
вүӨ-elimProp Pprop Pz Ps Pu Pl Pt (sвүӨs ОұвүӨОІ)   = Ps ОұвүӨОІ $ вүӨ-elimProp Pprop Pz Ps Pu Pl Pt ОұвүӨОІ
вүӨ-elimProp Pprop Pz Ps Pu Pl Pt (вүӨl ОұвүӨf)    = Pu ОұвүӨf $ вүӨ-elimProp Pprop Pz Ps Pu Pl Pt ОұвүӨf
вүӨ-elimProp Pprop Pz Ps Pu Pl Pt (lвүӨ {f} fвүӨ) = Pl fвүӨ
  (О» n вҶ’ вүӨ-elimProp Pprop Pz Ps Pu Pl Pt (fвүӨ n))
  (О» n вҶ’ вүӨ-elimProp Pprop Pz Ps Pu Pl Pt (snd f n))
вүӨ-elimProp Pprop Pz Ps Pu Pl Pt (вүӨ-trans ОұвүӨОІ ОІвүӨОі) = Pt
  (вүӨ-elimProp Pprop Pz Ps Pu Pl Pt ОұвүӨОІ)
  (вүӨ-elimProp Pprop Pz Ps Pu Pl Pt ОІвүӨОі)
вүӨ-elimProp Pprop Pz Ps Pu Pl Pt (isPropвүӨ ОұвүӨОІ ОІвүӨОі i) = isPropвҶ’PathP
  (О» i вҶ’ Pprop (isPropвүӨ ОұвүӨОІ ОІвүӨОі i))
  (вүӨ-elimProp Pprop Pz Ps Pu Pl Pt ОұвүӨОІ)
  (вүӨ-elimProp Pprop Pz Ps Pu Pl Pt ОІвүӨОі) i
```

## Derived order notations

The strict order is `Оұ < ОІ := suc Оұ вүӨ ОІ`, and we set up the usual negated and
flipped notations, plus a couple of small numerals.

```agda

_вүў_ : Rel Ord Ord _
Оұ вүў ОІ = В¬ Оұ вүЎ ОІ

_вүҘ_ : Rel Ord Ord _
Оұ вүҘ ОІ = ОІ вүӨ Оұ

_вү°_ : Rel Ord Ord _
Оұ вү° ОІ = В¬ Оұ вүӨ ОІ

_вүұ_ : Rel Ord Ord _
Оұ вүұ ОІ = В¬ ОІ вүӨ Оұ

_<_ : Rel Ord Ord _

_вү®_ : Rel Ord Ord _
Оұ вү® ОІ = В¬ Оұ < ОІ

_вүҜ_ : Rel Ord Ord _
Оұ вүҜ ОІ = В¬ ОІ < Оұ
```

## Basic order properties

Reflexivity is the first real use of `elimProp`: it follows the constructors,
with the limit case combining `lвүӨ` and `вүӨl`. From it we get that equal ordinals
are related, that every sequence element is `вүӨ` its limit (`fвүӨl`, the upper-bound
property as a closed term), and the one inversion that needs *no* encode-decode:
from `lim f вүӨ Оұ` we read back `f [ n ] вүӨ Оұ` just by transitivity with `fвүӨl`.

```agda
вүӨ-refl : isRefl' _вүӨ_
вүӨ-refl = elimProp (О» Оұ вҶ’ isPropвүӨ {Оұ} {Оұ}) zвүӨ sвүӨs (О» ih вҶ’ lвүӨ О» n вҶ’ вүӨl (ih n)) _

вүЎвҮ’вүӨ : Оұ вүЎ ОІ вҶ’ Оұ вүӨ ОІ
вүЎвҮ’вүӨ ОұвүЎОІ = subst (_вүӨ _) (sym ОұвүЎОІ) вүӨ-refl

fвүӨl : f [ n ] вүӨ lim f
fвүӨl = вүӨl вүӨ-refl

lвүӨ-inv : lim f вүӨ Оұ вҶ’ вҲҖ n вҶ’ f [ n ] вүӨ Оұ
lвүӨ-inv lвүӨОұ n = вүӨ-trans fвүӨl lвүӨОұ
```

## `вүӨ`-reasoning

Equational-style reasoning combinators for chaining `вүӨ`, `<`, and `вүЎ` steps;
these make the limit proofs below (and throughout the library) readable.

